# Implement a Laravel Feature

Use this skill for a new or expanded feature.

## Procedure
1. Restate the business outcome and acceptance boundaries.
2. Trace routes/controllers/services/models/migrations/UI/tests involved.
3. Find the closest existing feature and reuse its conventions.
4. Define state transitions, permissions, validation, response changes, data migration, side effects, and failure paths.
5. Create a file-by-file plan using `templates/implementation-plan.md`.
6. Implement the smallest vertical slice.
7. Add tests for success, invalid input, unauthorized access, and high-risk failure cases.
8. Review queries, transactions, indexes, idempotency, logging, and backward compatibility.
9. Run targeted tools and report results.
